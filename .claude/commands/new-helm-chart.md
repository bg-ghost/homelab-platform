description: Scaffold a new Helm chart following project conventions
allowed-tools: Bash(helm:*), Bash(mkdir:*), Bash(cp:*), Read, Write
---

Scaffold a new Helm chart named $ARGUMENTS under `helm-charts/$ARGUMENTS/`.

Follow project conventions:
1. Run `helm create helm-charts/$ARGUMENTS` as a starting point
2. Replace the default nginx templates with a minimal, opinionated set:
   - Deployment with resource limits (required by Kyverno policy)
   - Service
   - ServiceMonitor for Prometheus scraping
   - NetworkPolicy (deny-all ingress by default, explicit egress)
   - HorizontalPodAutoscaler (disabled by default via values)
3. Add to `values.yaml`:
   - `image.repository`, `image.tag`, `image.pullPolicy`
   - `resources.requests` and `resources.limits` (with sane defaults)
   - `podSecurityContext.runAsNonRoot: true`
   - `containerSecurityContext.readOnlyRootFilesystem: true`
4. Add a `README.md` with parameter table generated by helm-docs format

Print the file tree of the created chart when done.
